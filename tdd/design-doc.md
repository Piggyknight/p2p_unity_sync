# P2P Unity Sync - 设计文档

## 项目背景

### 问题现状
- **团队规模**: 200 开发者
- **资产规模**: SVN 资产 ~500GB，50-60 万散文件
- **开发场景**: 开放世界游戏，单次地形修改 ~1GB（约 5,000 个小文件）
- **痛点**:
  - SVN 中心服务器网络压力过大
  - 更新速度慢，影响开发效率
  - Unity Accelerator 也是集中式，进一步加剧压力

### 解决目标
1. **Phase 1**: P2P 局域网资源同步，分担 SVN 服务器压力
2. **Phase 2**: P2P 优化 Unity Library 与 Accelerator 2

### 环境约束
- 所有开发者在同一办公室局域网（单一局域网）
- 当前工作流: SVN + 集中式 Unity Accelerator
- Git 用于代码，SVN 用于资产

## 核心设计决策

### 1. 文件打包策略（Bundle）

**问题**: 不是单个大文件，而是海量小文件（1GB = 5,000 个 ~200KB 文件）

**方案**: 内容寻址包（Content-addressed Bundles）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Bundle Format                                   │
│                                                                              │
│   ┌──────────────────────────────────────────────────────────────────────┐  │
│   │                        Bundle Header                                   │  │
│   │   - Bundle ID (SHA256 of content)                                     │  │
│   │   - Bundle Name (e.g., "Assets_Scenes_Level1")                        │  │
│   │   - File Count                                                          │  │
│   │   - Total Size                                                           │  │
│   │   - Compression Algorithm                                               │  │
│   └──────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│   ┌──────────────────────────────────────────────────────────────────────┐  │
│   │                        File Manifest                                   │  │
│   │   ┌──────────────────┬─────────────┬──────────────┬─────────────────┐  │  │
│   │   │ File Path        │ File Size   │ Offset       │ SHA256          │  │  │
│   │   ├──────────────────┼─────────────┼──────────────┼─────────────────┤  │  │
│   │   │ Terrain.obj      │ 245KB       │ 0            │ a1b2c3...       │  │
│   │   │ Tree1.prefab      │ 12KB        │ 245000       │ d4e5f6...       │  │
│   │   │ ...              │ ...         │ ...          │ ...             │  │  │
│   │   └──────────────────┴─────────────┴──────────────┴─────────────────┘  │  │
│   └──────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│   ┌──────────────────────────────────────────────────────────────────────┐  │
│   │                        File Data (Compressed)                          │  │
│   │   [File 1][File 2][File 3]...                                         │  │
│   └──────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│   ┌──────────────────────────────────────────────────────────────────────┐  │
│   │                        Bundle Trailer                                  │  │
│   │   - Bundle SHA256 (for verification)                                  │  │
│   └──────────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────────┘
```

**打包策略**:
- 按目录结构打包（如 `Assets/Scenes/Level1/` → `Assets_Scenes_Level1.bundle`）
- 固定大小上限（如 100MB），超限时拆分
- 压缩算法：zstd（速度与压缩率平衡）

### 2. SVN Proxy 设计

**核心思想**: 拦截 SVN HTTP 请求，元数据透传，内容重定向到 P2P

```
svn update
    │
    ▼
┌─────────────────┐
│   SVN Proxy     │  (本地 HTTP 代理，端口 8080)
└────────┬────────┘
         │
         ▼
    分析请求
         │
         ├──► PROPFIND (目录/元数据) ──► 直接转发给 SVN
         │
         └──► GET (文件内容) ──► 拦截，路由到 P2P
                                        │
                                        ▼
                                   1. 查询 Tracker
                                   2. 获取 Peer 列表
                                   3. 并行 P2P 下载
                                   4. 验证完整性
                                   5. 写入工作副本
                                   6. 返回给 SVN 客户端
```

**Proxy 配置**:
```yaml
# svn-proxy-config.yaml
proxy:
  listen_port: 8080
  svn_server: http://svn-server.example.com

p2p:
  tracker_url: http://tracker-server:9000
  local_port: 8467
  max_parallel_downloads: 10
  timeout: 30

cache:
  metadata_ttl: 300
  content_dir: /var/cache/svn-p2p
  max_size: 50GB

fallback:
  enable_svn_fallback: true
  svn_timeout: 60
```

### 3. 双向工作流（Commit + Update）

**Commit 流程**:
```
开发者修改文件
     │
     ▼
svn commit ──► SVN Server (必须)
     │
     │ commit 成功，获得新版本号
     │
     ▼
本地打包新文件成 bundle
     │
     ▼
通知 Tracker: "我有 r12345"
     │
     ▼
局域网广播，其他开发者可 P2P 下载
```

**Update 流程**:
```
svn update
     │
     ▼
SVN Proxy 拦截
     │
     ├──► 获取元数据（变更列表）───────► SVN Server
     │
     └──► 查询 Tracker: 谁有这些版本?
     │              │
     │              ▼
     │         Peer 列表
     │              │
     ▼              ▼
   并行 P2P 下载 bundles
     │
     ▼
解包到正确位置
     │
     ▼
更新 SVN 元数据
```

### 4. 独立 Tracker 服务器

**职责**:
- Peer 注册与心跳（在线状态）
- Bundle 分布索引（谁有哪个 bundle）
- Revision 映射（哪些 peer 有完整版本）

**Tracker 数据结构**:
```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Central Tracker Server                              │
│                                                                              │
│  Peer Registry:                                                              │
│  ┌──────────────┬─────────────┬──────────────┬──────────────────────────┐     │
│  │ Peer ID      │ IP          │ Last Seen    │ Available                │     │
│  │              │             │              │ Revisions/Bundles        │     │
│  └──────────────┴─────────────┴──────────────┴──────────────────────────┘     │
│                                                                              │
│  Bundle Index:                                                                │
│  ┌──────────────────┬──────────────────────┬──────────────────────────┐     │
│  │ Bundle Hash      │ Bundle Name           │ Peers (who has it)       │     │
│  └──────────────────┴──────────────────────┴──────────────────────────┘     │
│                                                                              │
│  Revision Map:                                                                │
│  ┌──────────────┬──────────────────────┬──────────────────────────────────┐ │
│  │ Revision     │ Bundles in Revision   │ Peers with full revision         │ │
│  └──────────────┴──────────────────────┴──────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Tracker API**:
```python
# Peer 注册（上线时调用）
POST /api/peer/register

# Peer 心跳（每 30 秒）
POST /api/peer/heartbeat

# 查询：谁有这个版本/文件？
POST /api/peer/locate

# 通知：我有新内容了（commit 后）
POST /api/bundle/announce
```
