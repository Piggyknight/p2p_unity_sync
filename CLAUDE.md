## OpenSpec Workflow

This repository uses the OpenSpec experimental workflow system. The workflow is configured in `openspec/config.yaml` with the following settings:

- **Language**: Simplified Chinese (简体中文) for all artifacts
- **Technical terms**: Kept in original English (code identifiers, commands, file paths, API names)
- **Schema**: spec-driven development

### Available Commands

| Command | Purpose |
|---------|---------|
| `/opsx:explore` | Enter explore mode for thinking through ideas |
| `/opsx:propose` | Propose a new change and generate all artifacts |
| `/opsx:apply` | Implement tasks from an OpenSpec change |
| `/opsx:archive` | Archive a completed change |
| `/opsx:sync` | Sync delta specs from a change to main specs |
| `/red-create` | Create confirmed Redmine subtasks under a parent issue |
